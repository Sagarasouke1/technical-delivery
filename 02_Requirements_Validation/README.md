# Requirements Validation

## Objetivo
Validar que los requerimientos sean claros, completos y técnicamente viables antes del análisis.

## Contenido

- Levantamiento de requerimientos
- Reglas de negocio
- Criterios de aceptación
- Checklist de validación

## Buenas Prácticas

- No avanzar si el requerimiento es ambiguo
- Validar consistencia con reglas de negocio
- Asegurar criterios de aceptación claros
- Detectar dependencias desde esta etapa

## Regla Clave

Un requerimiento mal definido genera retrabajo; esta fase es obligatoria
