# User Stories

## Objetivo
Estructurar los requerimientos en unidades de trabajo claras y trazables.

## Contenido

- Backlog de historias de usuario
- Plantillas de historias

## Documentos esperados

- USR_User_Story_Backlog.xlsx
- USR_User_Story_Template.docx

## Buenas Prácticas

- Toda historia debe tener criterios de aceptación
- Debe ser entendible sin contexto adicional
- Debe estar ligada a un requerimiento
- Evitar historias demasiado grandes

## Regla Clave

No existe desarrollo sin historia de usuario definida
