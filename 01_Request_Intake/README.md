# Request Intake

## Objetivo
Centralizar todas las solicitudes iniciales antes de cualquier análisis técnico.

## Contenido

- Nuevas funcionalidades
- Solicitudes de cambio
- Reportes de hallazgos/incidencias
- Soluciones documentadas de hallazgos

## Documentos esperados

- FRM_New_Feature_Request.docx
- RFC_Change_Request.docx
- FRM_Finding_Report.docx
- FRM_Finding_Solution_Report.docx

## Buenas Prácticas

- Toda solicitud debe estar documentada antes de avanzar
- No se aceptan solicitudes verbales o sin evidencia
- Cada documento debe tener identificador único
- Debe incluir responsable y fecha

## Regla Clave

Ningún desarrollo inicia sin existir un documento en esta carpeta
